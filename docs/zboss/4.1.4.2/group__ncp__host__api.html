◆ ZB_ERROR_NCP_PROTOCOL_VERSION_MISMATCH

#define ZB_ERROR_NCP_PROTOCOL_VERSION_MISMATCH   1

This error is raised if current NCP protocol version of the Host does not match with the NCP FW protocol version of the SoC. NCP protocol version compatibility is checked at the Host startup.

◆ ZB_ERROR_NCP_TX_FAILED

#define ZB_ERROR_NCP_TX_FAILED   2

This error is raised if transport layer failed to transmit a packet, e.g. if the number of packet transmission attempts has been exceeded.

◆ ncp_host_custom_register_ind_cb()

void ncp_host_custom_register_ind_cb ( zb_ncp_custom_indication_cb_t  cb)

Registers a callback for processing the custom indication.

Parameters
cb- the indication callback
Note
Custom indications are user-defined and can vary. It's up to user to handle them. The indication payload is passed as a buffer payload.

Unless callback is registered or callback argument is NULL, the custom indication will be ignored.

)

Sends a custom request.

Parameters
param- The buffer filled with data to be sent. It's freed by the stack.
cb- Called with the returned payload, if NCP SoC replies with the zb_ncp_custom_response. Otherwise, only the status and TSN fields are valid and payload should not be read.
Note

Returns
tsn on success, NCP_TSN_RESERVED otherwise
